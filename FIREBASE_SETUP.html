Complete Setup Guide

ATTACH DATABASE & AUTHENTICATION

Your Firebase project is already created and connected. Follow these 5 steps to enable the database and login system. Takes about 10 minutes. No coding needed.

✅ ALREADY DONE: Your Firebase project (rrgarments) is created and the config keys are already pasted into js/firebase-auth.js. You don't need to edit any code. Just do the 5 steps below in the Firebase Console.

QUICK NAVIGATION

  1. Enable Email/Password Login
  2. Create the Database (Firestore)
  3. Set Database Security Rules
  4. Create Your Admin Account
  5. Import Starter Products

1ENABLE EMAIL/PASSWORD LOGIN

This lets customers create accounts and log in to your website.

  1. Go to console.firebase.google.com and click your project rrgarments
  2. In the left sidebar, click "Authentication" (it has a person icon)
  3. Click the "Sign-in method" tab at the top
  4. Click "Email/Password" in the list
  5. Toggle the first switch to "Enable" (turn it blue)
  6. Leave "Email link (passwordless)" OFF
  7. Click "Save" at the bottom
Done? You should see "Email/Password" with a green "Enabled" badge in the sign-in providers list.

2CREATE THE DATABASE (FIRESTORE)

This is where your products and customer orders are stored. It's Google's free cloud database.

  1. In the left sidebar, click "Firestore Database" (it has a database/cylinder icon)
  2. You'll see a message saying "Cloud Firestore consists of..." — click "Create database"
  3. Location: Choose asia-south1 (Mumbai) — this is closest to India and gives fastest speeds
  4. Click "Next"
  5. Security rules: Choose "Start in production mode" (this is secure — we'll set custom rules in Step 3)
  6. Click "Create"
  7. Wait 1–2 minutes (a loading spinner will appear). The database will be ready when you see the "Data" tab with empty collections.
Important: If you accidentally chose "test mode", that's okay — we'll fix the security rules in Step 3.

3SET DATABASE SECURITY RULES

This is critical. These rules control who can read/write your data. Without this, products won't save and orders won't be stored.

  1. In Firestore Database, click the "Rules" tab (next to "Data")
  2. You'll see some default code. Delete all of it.
  3. Paste this exact code in its place:
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // Products: anyone can view, only logged-in users can add/edit
    match /products/{productId} {
      allow read: if true;
      allow write: if request.auth != null;
    }

    // Orders: anyone can place an order, only logged-in users can view/manage
    match /orders/{orderId} {
      allow read: if request.auth != null;
      allow create: if true;
      allow update, delete: if request.auth != null;
    }
  }
}
  1. Click "Publish" button (top right of the code editor)
What these rules do:
• products — Anyone can see products on your store. Only you (logged in) can add/edit/delete them from the admin panel.
• orders — Customers can place orders without logging in (important for checkout). Only you can view and manage orders in the admin panel.
If you skip this step: Products won't save from the admin panel, and orders won't be stored when customers checkout. The website will fall back to showing products from the static JSON file.

3.5ENABLE IMAGE UPLOADS (FIREBASE STORAGE)

This lets you upload product images directly from your phone/computer in the admin panel. Images are stored in Google's cloud.

  1. In the left sidebar, click "Storage" (it has a folder/image icon)
  2. Click "Get started"
  3. Click "Next" (location will match your Firestore database)
  4. Click "Done"
  5. Now click the "Rules" tab (next to "Files")
  6. Delete the existing code and paste this:
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {

    // Product images: anyone can view, only logged-in users can upload
    match /products/{allPaths=**} {
      allow read: if true;
      allow write: if request.auth != null;
    }
  }
}
  1. Click "Publish"
What this does: Anyone visiting your store can see product images. Only you (logged in) can upload new images from the admin panel.
Alternative (no Storage setup): If you skip this step, you can still add products — just click "USE URL INSTEAD" in the image field and paste an image link (from Imgur, Google Drive, or your hosting's images/ folder). But enabling Storage is recommended for easy uploading from your phone.

4CREATE YOUR ADMIN ACCOUNT

This is the email and password you will use to log into the admin panel and manage your store.

  1. Open your website and go to the Account page (click the person icon in the header, or go to yourwebsite.com/account.html)
  2. Click the "SIGN UP" tab (top right of the form)
  3. Enter your email address (use a real email you check — e.g., owner@gmail.com)
  4. Enter a password (at least 6 characters — make it strong!)
  5. Confirm the password
  6. Click "CREATE ACCOUNT"
  7. You'll be redirected to the home page — you're now logged in!
Write down your email and password! This is your admin login. There is no "forgot password" on the website — if you forget it, you can reset it from the Firebase Console (Authentication → Users → click the user → Reset password).
✅ That's it! Your account is now created. Any logged-in user can access the admin panel. To restrict it to only your email, see the "Optional: Lock Admin Access" section below.

5IMPORT STARTER PRODUCTS & TEST

Your database is empty right now. Let's add your 3 starter products with one click, then verify everything works.

  1. Go to yourwebsite.com/admin.html
  2. Sign in with the email + password you just created
  3. You'll see the dashboard with stats (all at zero) and an empty products table
  4. Click the "IMPORT STARTER PRODUCTS" button (orange/olive button, left of "+ ADD PRODUCT")
  5. Click "OK" on the confirmation popup
  6. Wait a few seconds — you'll see a success message saying "3 products imported"
  7. The products table now shows your 3 items, and they instantly appear on your store!
To test the full flow:
1. Go to your store → add a product to cart → checkout → place order
2. Go back to admin panel → click the "ORDERS" tab
3. You'll see the order there with customer details!

OPTIONAL: LOCK ADMIN ACCESS TO YOUR EMAIL ONLY

Right now, anyone who creates an account can access the admin panel. To restrict it to only your email:

  1. Open the file js/admin.js in a text editor
  2. Find line 24 (near the top), which says:
    var ADMIN_EMAILS = []; // empty = any logged-in user is admin
  3. Change it to:
    var ADMIN_EMAILS = ["your-email@gmail.com"];
  4. Replace your-email@gmail.com with the email you signed up with
  5. Save the file and upload it to your hosting
Now only your email can access the admin panel. Other users can still create accounts and shop, but they can't see the admin dashboard.

FREQUENTLY ASKED QUESTIONS

Q: Do I need to edit any code?

A: No. Your Firebase credentials are already in the code. You only need to do the 5 steps above in the Firebase Console (a website). No coding or file editing required.

Q: Is Firebase free?

A: Yes. The free "Spark" plan includes 50,000 database reads, 20,000 writes, and 1GB storage per day. That's enough for hundreds of orders per day. You only pay if you get very high traffic.

Q: Where are orders stored?

A: In Firebase Firestore (Google's cloud). When a customer checks out, the order is instantly saved to the orders collection. You see it in the admin panel under "ORDERS".

Q: How do I add new products after setup?

A: Go to yourwebsite.com/admin.html → sign in → click "+ ADD PRODUCT" → fill the form (name, price, image, sizes, colors) → click "SAVE PRODUCT". It appears on your store instantly. No code editing.

Q: How do I add product images?

A: Upload image files to your hosting's images/ folder (via FTP/cPanel/Netlify). Then in the admin panel, type the path like images/my-product.jpg. Or use a full URL from any image host (Imgur, Google Drive, etc.). Keep images under 200KB — use tinypng.com to compress.

Q: Can I manage the store from my phone?

A: Yes! The admin panel works on any phone browser. Just go to yourwebsite.com/admin.html and log in.

Q: What if I forget my admin password?

A: Go to Firebase Console → Authentication → Users → find your email → click "Reset password". Firebase will send a password reset email.

Q: How do I accept payments?

A: This setup captures orders but doesn't process payments online. For payments, you can:
• Use UPI/WhatsApp — customer places order, you message them your UPI ID (simplest, free)
• Integrate Razorpay (supports UPI, cards, net banking — best for India, 2% transaction fee)
• Integrate Stripe (international cards)
The order is always captured regardless, so you can contact the customer to arrange payment.

Q: What happens if two customers order the same product?

A: The stock count in the admin panel doesn't auto-decrease yet. You'll need to manually update stock after fulfilling orders. (Auto stock management can be added later.)

  4. Your website → Account page → Sign Up with your email + password
  5. Your website → admin.html → Sign in → Click "IMPORT STARTER PRODUCTS"
That's it! Your store is now fully functional with cloud database, authentication, and order management.
